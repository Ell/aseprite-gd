# AGENTS.md — aseprite-gd

Native Aseprite (`.aseprite`/`.ase`) importer for Godot 4, written in pure Rust
as a GDExtension. Parses the binary format directly — **never** shells out to
the Aseprite CLI at import time (that constraint is the entire reason this
project exists).

## Workspace

```
crates/ase-core      pure parser + compositor — ZERO godot deps, publishable to crates.io
crates/ase-cli       dev binary (`ase`) — inspect files, render frames, generate golden images
crates/aseprite-gd   gdext cdylib — importers, runtime loader, Godot-type conversion only
godot/               demo/test Godot project; loads the built extension from ../target
```

`cargo build` / `cargo test` default to the fast crates (ase-core, ase-cli).
The gdext crate is excluded from default-members because compiling `godot`
takes minutes: build it explicitly with `cargo build -p aseprite-gd`.

## Commands

```sh
cargo test                          # ase-core unit tests — the tight loop
cargo build -p aseprite-gd          # build the GDExtension (slow first time)
cargo run -p ase-cli -- info FILE   # inspect an .aseprite file
cargo clippy --workspace --all-targets -- -D warnings
cargo fmt --all
```

Integration check (requires a `godot` binary, 4.2+):

```sh
godot --headless --path godot --import   # must succeed with zero external deps
```

## Hard rules

1. **`ase-core` must never depend on `godot`** (or any engine/framework crate).
   Anything reusable outside Godot goes in `ase-core`; `crates/aseprite-gd`
   only adapts. If you're importing Godot types into core, stop.
2. **No `unsafe` in parse paths.** All input access goes through
   `ase_core::read::Reader` (bounds-checked). Input files are untrusted —
   mods, downloads, fuzzers.
3. **Enforce `ase_core::limits`** on anything size-like from the file:
   decompression output, palette sizes, recursion depth. Reject, don't clamp.
4. **Seek by declared sizes, never by consumption** (spec gotcha #1). After a
   chunk: `seek(chunk_start + chunk_size)`. After a frame:
   `seek(frame_start + frame_bytes)`. Unknown chunks/fields are skipped
   silently — that's the format's forward-compatibility mechanism.
5. **Pixel-exact means Aseprite's exact integer math.** Blend modes use
   `MUL_UN8`/`DIV_UN8` and the `_n` new-blend variants from the reference §9 —
   do not "simplify" to floating point; correctness is verified against
   Aseprite's own renders in the golden suite.
6. **Deterministic output.** Same input bytes → same output bytes (stable
   iteration/packing order, no HashMap-order-dependent results). CI imports
   must be reproducible across machines.
7. **Structured errors with offsets.** Every `ParseError` carries the absolute
   file offset. No `panic!`/`unwrap` on data derived from file contents.

## gdext (godot-rust) specifics

- The `godot` crate is pinned to 0.5.x with `api-4-2` (minimum supported Godot)
  and **`experimental-threads`** — the editor calls `ResourceFormatLoader`
  methods from background threads (gdext#597). Keep loader/importer state
  `Send`-safe and minimal.
- Editor classes need `#[class(tool)]`. `EditorPlugin` subclasses auto-register.
- Don't touch `Gd<Self>` in `init()` — it panics (gdext#997); wire signals in
  `enter_tree()`/`ready()`.
- Implement `get_priority`/`get_import_order` explicitly on import plugins —
  engine defaults are unreliable (godot#104519).
- Panics are caught at the FFI boundary and print as Godot errors; they won't
  crash the editor, but treat any panic reachable from file data as a bug.
- Hot reload (`reloadable = true`) works but tool classes re-run `ready()` on
  reload — guard side effects. macOS reload is flaky; editor restart is the
  documented fallback.

## Testing expectations

- New parse code lands with unit tests using hand-built byte fixtures (see
  `parse/header.rs` tests for the pattern) — including at least one truncation
  and one invalid-value case.
- Compositing code lands with golden-image comparisons once the corpus exists
  (fixtures under `crates/ase-core/tests/`; generated by a dev-only script
  that drives real Aseprite — the CLI is fine *there* because it never ships).
- Parser changes should keep `cargo fuzz` targets building
  (`crates/ase-core/fuzz`, once added).

## Style

- Comments cite the spec (`§6.11`, `gotcha #17`) instead of re-explaining the
  format. If you discover behavior the reference doesn't cover, update
  docs/ase-format-reference.md in the same change.
- Conventional commits (`feat:`, `fix:`, `docs:`, `test:`, `refactor:`). Terse,
  lowercase subjects; body only when the why isn't obvious.
